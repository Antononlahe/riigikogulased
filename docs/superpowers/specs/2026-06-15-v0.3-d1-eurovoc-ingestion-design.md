# v0.3 / D1 — Eurovoc topic ingestion + taxonomy + topic-filtered discipline foundation (design)

**Date:** 2026-06-15
**Status:** approved approach, spec under review
**Scope:** The first half of v0.3 (Topics). Ingest the official Eurovoc subject taxonomy and
link **votes → bills → subjects**, with the full field/microthesaurus hierarchy, and lay the
SQL foundation for topic-filtered discipline. **No UI** — D1 is `apps/scraper` + a migration +
a view, verified by SQL. The topic UI (filter chips, badges, facets, topic-filtered discipline
display) is **D2**.

Builds on the existing schema: `votes.draft_uuid` (each voting's `relatedDraft`, populated in
A2) is the bridge to bills. Discipline numbers are unchanged.

---

## Why

Governing decision 3 of the roadmap: topic categorization uses **official Eurovoc tags**, not
manual rules or an LLM. The Riigikogu API carries them per bill. v0.3 makes "filter discipline
by topic" and per-MP topic facets possible; D1 lands the data + the query foundation so D2 is
purely additive UI.

## Feasibility (verified live, 2026-06-15)

- **`/api/volumes/drafts/{uuid}`** returns the draft (bill) with a **`descriptors`** array:
  `[{edid, text}, …]` (e.g. bill 866 → `haldusmenetlus`, `e-valitsus`, `andmetöötlus`, …).
  `edid` is the stable Eurovoc descriptor id. (Note: the generic `/api/volumes/{uuid}` does
  NOT carry descriptors — the **draft** endpoint does.)
- **`/api/eurovoc/fields?lang=`** returns the 21 top-level fields, each with its
  `microThesauruses` (`etid`, `code`, `text`; descriptor lists lazy/empty here).
- **`/api/eurovoc/microthes?etid=&lang=`** returns a microthesaurus with its `descriptors`.
- Hierarchy: **field (`efid`) → microthesaurus (`etid`) → descriptor (`edid`)**. (The
  per-descriptor `descriptor?edid=`/`descriptorHierPaths` rollup is unreliable — empty for the
  sample — so the hierarchy is built top-down from fields + microthes.)
- Coverage: **395 / 598 votes** carry a `draft_uuid` (233 distinct drafts); the rest are
  procedural/non-bill votes with no topic (correct).
- All endpoints are the same `api.riigikogu.ee`, **1 req/s** (429 above), CC-BY-SA.

## Locked decisions (from brainstorming, 2026-06-15)

1. **Two slices:** D1 = ingestion + model + topic-filtered-discipline foundation (this spec);
   D2 = topic UI.
2. **Full taxonomy with hierarchy:** ingest all 21 fields + their microthesauruses +
   descriptors (top-down), so fine descriptors roll up into broad domains for clean faceting.
3. **Link topics to `draft_uuid` directly** — no `volumes`/`bills` table yet (that's v0.6).
   Votes already carry `draft_uuid`/`draft_title`/`draft_mark`.
4. **Bilingual labels** (et + en) for fields/microthesauruses/descriptors (each taxonomy
   endpoint fetched once per language); ET is authoritative.

## Schema — migration `0005_eurovoc.sql`

```sql
CREATE TABLE eurovoc_fields (
  efid    INT PRIMARY KEY,           -- Eurovoc field id
  uuid    UUID NOT NULL,
  code    TEXT,
  text_et TEXT NOT NULL,
  text_en TEXT
);
CREATE TABLE eurovoc_microthesauri (
  etid       INT PRIMARY KEY,        -- microthesaurus id
  uuid       UUID NOT NULL,
  code       TEXT,
  text_et    TEXT NOT NULL,
  text_en    TEXT,
  field_efid INT REFERENCES eurovoc_fields(efid)
);
CREATE TABLE eurovoc_descriptors (
  edid              INT PRIMARY KEY, -- descriptor id (as carried on bills)
  uuid              UUID,
  code              TEXT,
  text_et           TEXT NOT NULL,
  text_en           TEXT,
  microthesaurus_etid INT REFERENCES eurovoc_microthesauri(etid)
);
CREATE TABLE volume_topics (
  draft_uuid     UUID NOT NULL,      -- = votes.draft_uuid (a bill)
  descriptor_edid INT NOT NULL REFERENCES eurovoc_descriptors(edid),
  PRIMARY KEY (draft_uuid, descriptor_edid)
);
CREATE INDEX volume_topics_descriptor_idx ON volume_topics (descriptor_edid);

-- Foundation view: a vote resolved to its bill's descriptors + the rollup.
CREATE VIEW vote_topics AS
SELECT v.id AS vote_id, vt.descriptor_edid,
       d.microthesaurus_etid, mt.field_efid
FROM votes v
JOIN volume_topics vt ON vt.draft_uuid = v.draft_uuid
JOIN eurovoc_descriptors d ON d.edid = vt.descriptor_edid
LEFT JOIN eurovoc_microthesauri mt ON mt.etid = d.microthesaurus_etid;
```

`member_discipline` / `member_vote_alignment` are unchanged. Topic-filtered discipline (D2) is
`member_vote_alignment ⋈ vote_topics` filtered by `field_efid`/`descriptor_edid`, then
aggregated the same way `member_discipline` aggregates — no new scoring logic.

## Ingestion — scraper `eurovoc` command + offline `rebuild`

New modules mirroring the existing `api_*` / `ariregister_*` pattern (reuse `ApiClient`'s 1
req/s throttle + backoff):
- `eurovoc_models.py` — pydantic models for the field/microthesaurus/descriptor + draft
  `descriptors` shapes; pure mappers (raw → rows).
- `eurovoc_cache.py` — git-committed raw-JSON archive under `cache/api/eurovoc/` (the fields
  tree per lang, each microthesaurus per lang) and the per-draft descriptor responses under
  `cache/api/drafts/`.
- writer functions (`write_eurovoc_taxonomy`, `write_volume_topics`) + `db` upserts.
- CLI command **`eurovoc`** (parallel to `members`/`erakond`/`photos`):
  1. `GET /api/eurovoc/fields?lang=et` + `lang=en` → `eurovoc_fields` + field↔microthesaurus.
  2. for each microthesaurus `GET /api/eurovoc/microthes?etid=&lang=et|en` → `eurovoc_microthesauri`
     (with `field_efid`) + `eurovoc_descriptors` (with `microthesaurus_etid`).
  3. for each distinct `votes.draft_uuid` `GET /api/volumes/drafts/{uuid}` → `descriptors`
     (edid) → `volume_topics`. A descriptor on a bill that is missing from the taxonomy is
     inserted from the draft's `{edid, text}` (microthesaurus NULL) so no link is dropped.
- `rebuild` replays the committed eurovoc + draft caches (offline), after the existing
  votings/members/erakond steps.

Politeness: ~258 taxonomy calls + 233 draft calls (~8 min once, then cached); CC-BY-SA
attribution; offline `rebuild` reproduces with no network.

## Foundation & validation

D1 ships the data + the `vote_topics` view, validated by SQL (per the roadmap check):
- a sample of votes resolves to a bill and **≥1 descriptor and a field** (spot-check);
- every `volume_topics.descriptor_edid` exists in `eurovoc_descriptors` (FK holds);
- **topic-filtered discipline reconciles**: for a chosen field, the topic-filtered counted is
  ≤ the unfiltered counted, and a member's union over all fields' counted equals their
  draft-vote counted (modulo multi-topic overlap, which is expected and noted);
- ~233 drafts resolve; report any draft that 404s or returns no descriptors.

## Testing

- **Offline unit tests** (pytest) for `eurovoc_models` pure mappers (fields/microthes/draft →
  rows, including a descriptor-not-in-taxonomy case) and the `eurovoc_cache` round-trip,
  against committed fixtures under `fixtures/api/eurovoc/`.
- Taxonomy + links + the `vote_topics` view validated on a Neon branch, then prod (additive:
  new tables + a view, no discipline change), cache committed. Same protocol as A1/A2/erakond.

## Cutover

Apply `0005`; run `eurovoc` live (caches the taxonomy + draft topics, writes the tables);
validate on a branch (the reconciliation checks above), then prod; commit the eurovoc/draft
cache. No discipline change.

## Out of scope → **D2**

All UI: topic-filter chips on the members list, topic badges (per vote / on the timeline),
topic-filtered discipline display, and per-MP topic facets. Also: `volumes`/`bills` tables and
sponsorship (v0.6); EU/interpellation/written-question volume topics (later); the Eurovoc
broader/narrower (`relatedTerms`) graph beyond the field/microthesaurus rollup.
